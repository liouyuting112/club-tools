# 🟡【工具一】情侶 & 家人大小事小本本

## 功能描述
一個多人協作的「重要的人喜好記錄簿」。
用戶可以建立多個人物檔案（另一半、爸爸、媽媽、貓貓等），
記錄每個人的喜好、討厭的事、生日、紀念日、照片，
並支援查詢功能。多個家庭成員可共同編輯同一份記錄。

## 核心功能清單

### 1. 人物檔案系統
- 建立人物卡片：姓名、關係（另一半 / 家人 / 寵物 / 朋友）、生日、頭像（可上傳圖片或選emoji）
- 每個人物卡片有獨立頁面
- 人物類型用不同顏色區分（另一半=粉紅、家人=橘、寵物=綠、朋友=藍）

### 2. 喜好記錄（分類標籤系統）
每個人物底下可新增條目，分類如下：
- ❤️ 喜歡的食物（例：草莓乾、珍珠奶茶）
- 🚫 不喜歡 / 地雷食物（例：香菜、苦瓜）
- 🎁 喜歡的禮物類型
- 😤 NG行為（例：不喜歡被催）
- 🎵 喜好（音樂、興趣等）
- 📅 重要日期（生日、紀念日、考試日）
- 📝 其他備註
- 用戶可自訂標籤類別

### 3. 協作機制（邀請共編）
- 建立「小本本空間」時，系統產生一組邀請碼（6位英數）
- 其他人輸入邀請碼即可加入，可共同查看 & 編輯
- 每個空間有「管理員」（建立者）可以：
  - 移除成員
  - 設定成員為「只能看」或「可以編輯」
- 不需要強制註冊，可用 nickname + 邀請碼進入（降低使用門檻）
- 有帳號者可綁定多個空間

### 4. 查詢功能
- 在空間內輸入自然語言查詢，例如：
  - 「媽媽喜歡吃什麼？」→ 列出媽媽的喜愛食物條目
  - 「誰的生日快到了？」→ 列出30天內的生日/紀念日
  - 「草莓乾」→ 搜尋所有包含此關鍵字的人物條目
- 搜尋用關鍵字比對即可，不需要 AI

### 5. 提醒功能（加分項）
- 生日/紀念日前 7 天，可選擇發 Email 提醒（不強制）

### 6. 流量統計
- 記錄每日活躍空間數、新建人物數、條目新增數
- 後台頁面（/admin）顯示累計使用人次、空間數

---

## 資安規範（重要！請確實遵守）

### 輸入驗證
- 所有輸入欄位做 server-side 驗證，過濾 XSS（HTML注入、Script注入）
- 圖片上傳：限制副檔名（jpg/png/webp/gif）、限制大小（最大 2MB）、存前重新命名（不用原始檔名）
- 不接受 .exe / .php / .js 等非圖片檔案

### 邀請碼安全
- 邀請碼有效期限：預設 7 天，可設定為永久或單次使用
- 進入空間後需設定 nickname，nickname 儲存在 session/cookie，不需要密碼登入也可使用

### 資料隔離
- 每個空間的資料完全隔離，不能跨空間查詢
- API 存取時驗證用戶是否屬於該空間的成員

### 防濫用
- 單一 IP 每小時最多建立 3 個新空間（防止惡意大量建立）
- 圖片上傳每小時每空間上限 20 張

### HTTPS
- 全站強制 HTTPS，不接受 HTTP 連線

---

## UI/UX 方向
- 風格：溫暖、手帳感、像一本小本本
- 主色：奶油色背景 + 珊瑚粉 + 抹茶綠
- 字型：圓潤可愛（Nunito / Noto Sans TC）
- 行動裝置優先（手機版為主，家人間用手機傳連結加入）
- 動畫：卡片翻頁感、新增條目時有小動畫

---

## 技術建議
- 前端：Vue 3 or React（component-based 好維護）
- 後端：Node.js (Express) or Python (FastAPI)
- 資料庫：PostgreSQL（人物/條目/空間）+ Redis（session）
- 圖片存儲：Cloudflare R2 or AWS S3（或直接存 base64 在 DB，小型工具可接受）
- 部署：Railway / Render / Vercel（前端）+ Railway（後端）

# 🟢【工具二】購物衝動冷靜器

## 功能描述
想衝動購物前，輸入商品名稱和價格，
工具會問你 3 個問題，最後給出「買 or 不買」的理性判斷。

## 核心功能

### 流程
1. 輸入：商品名稱 + 價格（必填）+ 可選填「為什麼想買」
2. 工具依序問 3 個問題（用按鈕選答，不用打字）：
   - Q1：「你上次想到它是多久前？」→ 選項：今天才看到 / 想了一週 / 想了一個月以上
   - Q2：「如果今天不買，你會難過多久？」→ 選項：一整天 / 幾小時 / 忘了就算了
   - Q3：「你這個月的預算還夠嗎？」→ 選項：很夠 / 勉強 / 其實不夠
3. 根據答案組合，給出結論（共 8 種結果組合）
   - 例：「今天才看到 + 忘了就算了 + 不夠」→ 「🚫 把錢留著，下個月還在賣的話再說」
   - 例：「想了一個月 + 難過一整天 + 很夠」→ 「✅ 買吧！你已經等夠久了」
4. 結論頁面有「分享結果」按鈕（產生圖片 or 文字可複製）

### 統計顯示
- 頁面底部顯示：「已經幫 X 個人冷靜下來 💸」（累計使用次數）
- 按「買」vs「不買」的比例圓餅圖（趣味數據）

---

## 資安規範
- 輸入欄做 XSS 過濾
- 不儲存個人資料，只存統計數字（買/不買次數、使用總次數）
- Rate limit：單一 IP 每分鐘最多 10 次送出（防機器人刷數據）

---

## UI/UX 方向
- 風格：衝動感 vs 冷靜感的視覺對比（購物袋icon vs 冰塊icon）
- 主色：橘紅（衝動）→ 藍綠（冷靜）的漸層轉換動畫
- 問答卡片有滑入/滑出動畫
- 結果頁面大字顯示，視覺衝擊強

# 🟢【工具三】Offer 比較計算機

## 功能描述
輸入兩個工作 Offer 的條件，自動計算實際差距，
幫你做理性決策。

## 核心功能

### 輸入欄位（Offer A / Offer B 兩欄對比）
- 月薪（必填）
- 年終（幾個月）
- 交通費（月）
- 餐費補貼（月）
- 年假天數
- 上班地點（輸入縣市，自動抓大概通勤成本）
- 可選填：遠端天數/週、試用期、保險

### 計算邏輯
- 年總收入 = (月薪 × 12) + (月薪 × 年終月數) + (補貼 × 12)
- 年成本 = 交通費 × 12（可輸入月交通費或自動估算）
- 實際年所得 = 年總收入 - 年成本
- 時薪 = 實際年所得 ÷ (工作天 × 8小時) → 扣掉年假更準確
- 顯示：兩者實際差距（每年差多少、每月差多少）

### 輸出
- 視覺化比較表格（左右並列，差距用顏色標示）
- 一句話總結：「Offer B 每年多賺 NT$XX,XXX，但年假少 5 天，換算時薪其實差不多」
- 可複製/截圖分享結果

### 統計
- 累計使用次數顯示
- 「最多人選 A 還是 B？」（匿名統計，趣味數據）

---

## 資安規範
- 純前端計算，不需要儲存用戶輸入的薪資數字到後端
- 只送出最終「選A/選B/無法決定」的匿名統計到後端
- Rate limit 保護統計 API

---

## UI/UX 方向
- 風格：乾淨、專業，像 Excel 但好看很多
- 左右分欄對比，差距用箭頭和顏色視覺化
- 行動裝置改為上下排列

# 🟢【工具四】你今天值得吃大餐嗎

## 功能描述
輸入今天做了什麼事，系統「認真評估」你今天的努力程度，
判斷你deserve幾星級的晚餐。（廢但好玩）

## 核心功能

### 流程
1. 文字輸入框：「說說你今天做了什麼？」（自由輸入，或點選預設選項）
2. 預設快選標籤（可多選）：
   - 上班了 / 開了很多會 / 被老闆罵 / 加班 / 擠捷運 / 健身 / 喝夠水 / 什麼都沒做 / 沒睡好 / 哭了一下 etc.
3. 「計算看看」按鈕
4. 系統顯示「計算中...」的假loading（加趣味感，1.5秒）
5. 輸出結論，格式如下：

```
⭐⭐⭐⭐ 四星評級

今日成就：擠了捷運、被老闆罵、還是撐完一整天
診斷：你今天承受了相當程度的社會壓力，
      搭配輕微的存在主義危機，
      判定值得吃一頓正式晚餐。

推薦：火鍋 or 燒肉
禁止：吃泡麵打發自己
```

### 判斷邏輯（純 if-else，不需要 AI）
根據關鍵字和選項組合，預設 20-30 種結論模板，
隨機抽取符合條件的模板顯示。
- 關鍵字「什麼都沒做」→ 「今天的你選擇了休息，休息也是一種勇氣，值得吃頓好的」
- 關鍵字「加班」「被罵」→ 四星以上
- 什麼都沒勾 → 一星，但結論超好笑

### 統計
- 「今天已有 X 人在評估自己值不值得吃大餐 🍽️」

---

## 資安規範
- 文字輸入做 XSS 過濾
- 不儲存用戶輸入的文字，只存統計次數
- Rate limit：單 IP 每分鐘最多 5 次

---

## UI/UX 方向
- 風格：超認真的假評分感（像米其林評鑑但很廢）
- 動畫：loading 時出現假進度條「分析勞動強度... 評估情緒損耗... 計算大餐資格...」
- 結果有大大的星星評級
- 分享按鈕（複製文字去貼給朋友）

# 🟡【工具六】寵物食物安全查詢

## 功能描述
輸入食物名稱，立刻告訴你狗或貓能不能吃、
注意事項、安全劑量（如果適量可吃的話）。

## 核心功能

### 資料庫建置（重點！）
- 建立「食物安全資料庫」（約 150-300 種常見食物）
- 每筆資料欄位：
  - 食物名稱（中文 + 英文 + 常見別名）
  - 狗：安全/危險/適量/未知
  - 貓：安全/危險/適量/未知
  - 危險原因（如含有xylitol、葡萄毒素等）
  - 症狀描述（吃了會怎樣）
  - 建議處理方式
  - 資料來源（美國ASPCA / VCA醫院等可信來源）

### 搜尋功能
- 輸入框 + 即時搜尋（輸入就顯示結果，不用按Enter）
- 同義詞搜尋：輸入「葡萄」也能找到「葡萄乾」「提子」
- 搜不到時顯示：「我們資料庫沒有這筆資料，建議詢問獸醫」（不亂猜）

### 輸出格式
```
🍫 巧克力

🐕 狗：⛔ 危險！
🐈 貓：⛔ 危險！

危險原因：含有可可鹼（Theobromine），狗貓均無法代謝
症狀：嘔吐、腹瀉、心跳加速、嚴重可致死
劑量：狗體重每公斤攝入 20mg 即可能中毒
緊急處理：立即就醫，告知獸醫攝入量和時間
```

### 統計
- 「今天已查詢 X 次，最常被查的食物是＿＿」
- 熱門查詢排行榜（有趣的附加功能）

---

## 資安規範
- 搜尋輸入做 XSS 過濾、SQL injection 防護
- Rate limit：單 IP 每分鐘最多 30 次查詢
- 資料庫唯讀（搜尋功能不允許任何寫入操作）
- 加入免責聲明：「本工具僅供參考，緊急情況請立即就醫」

---

## UI/UX 方向
- 主色：乾淨白底 + 綠色（安全）/ 紅色（危險）/ 黃色（適量）
- 搜尋框大、好點擊，手機版友好
- 危險食物用紅色醒目標示，視覺清楚

# 🟢【工具七】今天要幾點睡才不算太晚（廢物系列）

你問它：「我明天幾點要起床？」

- 「今天有 X 個人在 12 點後問這個問題（都來不及了）」

# 🔴【工具八】替死鬼連結預覽器

## 功能描述
輸入可疑連結，由伺服器代替你去訪問，
回傳截圖 + 最終真實網址（解析縮網址跳轉）。

## 核心功能
1. 輸入框貼入任何網址（bit.ly / 可疑連結 / 不明來源）
2. 後端用 headless browser（Puppeteer）去訪問
3. 回傳：
   - 網站截圖（720px 寬）
   - 最終落地網址（跟蹤所有跳轉）
   - 跳轉次數
   - 是否包含已知釣魚/惡意網址（比對黑名單資料庫）
4. 截圖旁邊顯示安全評估：✅ 看起來正常 / ⚠️ 請謹慎 / 🚫 已知惡意網站

## 資安規範（本工具特別重要！）
- **沙箱隔離**：Puppeteer 必須在 Docker 容器或獨立沙箱環境執行，不能與主應用同一環境
- **禁止訪問內網**：訪問前先解析 IP，拒絕 192.168.x.x / 10.x.x.x / 127.x.x.x / 169.254.x.x（防止 SSRF 攻擊）
- **禁止 file:// 協議**：只允許 http:// 和 https:// 網址
- **超時設定**：單次訪問最多 10 秒，超時即停止
- **截圖大小限制**：壓縮至 200KB 以內再回傳
- **Rate limit**：單 IP 每小時最多 20 次查詢（headless browser 資源消耗大）
- **Queue 系統**：訪問請求排隊處理，防止同時大量請求拖垮伺服器
- **不儲存截圖**：截圖產生後直接回傳，不存到伺服器
- **黑名單比對**：可整合 Google Safe Browsing API（免費額度夠用）

# 📊 共用規範（所有工具都要做）

## 流量統計模組
每個工具都要在後台記錄：
- 累計使用次數（顯示在前台，增加可信度）
- 每日 UV / PV
- 來源（直接進入 / 社群分享）
- 使用裝置（手機/桌機比例）

## 分享功能
- 每個工具都要有「分享」按鈕
- 產生可複製的文字結果 or 截圖（用 html2canvas）
- 社群分享連結（FB / LINE / 複製連結）

## 通用資安規範
- 全站 HTTPS
- 所有 API 加 rate limiting（建議用 express-rate-limit or 等效工具）
- 所有輸入欄位做 server-side XSS 過濾（不能只靠前端）
- SQL 查詢全部用 parameterized query（防止 SQL injection）

- 定期備份資料庫
- 上線前跑一次 OWASP ZAP 或類似的基本安全掃描
